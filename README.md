# Intern Attendance Tracking Web Application

Complete mobile-friendly attendance app built with Angular 11, Angular Material, TypeScript, RxJS, Reactive Forms, HttpClient, Node.js, Express.js, MongoDB, Multer, and node-cron.

The QR code is not generated by this app. The existing QR code only acts as an entry point to the scanner or attendance portal.

## What You Will Build

- First-time intern registration with name and profile photo.
- Auto-generated Intern IDs in `INT001`, `INT002`, `INT003` format.
- Local storage based registration guard so the registration page does not appear again.
- QR scanner page using the mobile camera.
- Attendance page with Login and Logout rules.
- MongoDB-backed intern and attendance records.
- 4-hour backend notification status job using `node-cron`.
- Admin dashboard with search, date filter, currently logged-in filter, timestamps, total hours, Excel export, and PDF export.

## Prerequisites From Scratch

Install these first:

1. Node.js. Recommended for Angular 11: Node.js 14.x or 16.x. Modern Node versions such as Node.js 17+ also work in this project because the frontend scripts set `NODE_OPTIONS=--openssl-legacy-provider`.
2. Angular CLI 11:

```bash
npm install -g @angular/cli@11
```

3. MongoDB Community Server, or MongoDB Atlas.
4. Git is optional, but useful.

Check versions:

```bash
node -v
npm -v
ng version
mongod --version
```

Open a terminal in `backend`.

## User Flow

1. Open `http://localhost:4200`.
2. First-time users are redirected to `/register`.
3. Enter name and upload a profile photo.
4. Backend generates an Intern ID such as `INT001`.
5. Frontend stores the Intern ID in browser local storage.
6. User scans the existing office QR code from `/scan`.
7. Valid scan redirects to `/attendance`.
8. User clicks Login.
9. User clicks Logout when leaving.
10. After logout, both attendance actions are disabled until the next day.

## QR Scanner

The scanner uses `@zxing/ngx-scanner`.

Valid QR contents currently include any value containing:

```text
attendance
/attendance
intern-attendance
```

You can change this in:

```text
frontend/src/app/features/scanner/scanner.component.ts
```

For production, make the existing QR code point to:

```text
https://your-domain.com/attendance
```

or to a string that matches your chosen validation token.

## Attendance Rules

- Before login: Login enabled, Logout disabled.
- After login: Login disabled, Logout enabled.
- After logout: Login disabled and Logout disabled until the next day.
- Duplicate login is blocked by the backend.
- Logout without active login is blocked by the backend.

## 4-Hour Notification Job

The backend runs a cron job every minute.

When an attendance record is at least 4 hours old and `notificationSent=false`:

- If `isLoggedIn=true`, it sets `notificationType="still_logged_in"`.
- If `isLoggedIn=false`, it sets `notificationType="logged_out"`.

The attendance page shows the notification through an Angular Material snackbar when the record is loaded.

## Common Problems

Camera does not open:

- Use HTTPS in production.
- On localhost, camera permission should work in modern browsers.
- Check browser camera permissions.

MongoDB connection fails:

- Confirm MongoDB is running.
- Confirm `MONGO_URI` in `backend/.env`.

Registration succeeds but image does not show:

- Confirm backend is running on port `5000`.
- Confirm `assetBaseUrl` in `frontend/src/environments/environment.ts`.

Dependencies fail to install:

- Best option: use Node 14 or 16 for Angular 11.
- Modern Node option: use the included `npm start` and `npm run build` scripts. They already apply `NODE_OPTIONS=--openssl-legacy-provider`.
- Delete `node_modules` and reinstall.

## Node.js and Angular 11 Compatibility

Angular 11 uses Webpack 4. Webpack 4 can fail on Node.js 17 and newer because those Node versions use OpenSSL 3.

This project keeps Angular 11, as requested, and fixes modern Node execution by adding `cross-env` and updating frontend scripts:

```json
{
  "start": "cross-env NODE_OPTIONS=--openssl-legacy-provider ng serve",
  "build": "cross-env NODE_OPTIONS=--openssl-legacy-provider ng build",
  "start:legacy-node": "ng serve",
  "build:legacy-node": "ng build"
}
```

Use these commands on your current Node.js v24:

```bash
cd frontend
npm install
npm start
```

If you install Node.js 14 or 16, you can also run:

```bash
npm run start:legacy-node
```
